Reconfigurable scan networks ABSTRACT  
The IEEE 1687 Std. (IJTAG) introduces an efficient access methodology based on reconfigurable scan networks to 
address the ever-increasing complexity of the latest system-on-chips. By invoking this new methodology, the 
overall test time can considerably be reduced by shortening the scan chains' length without sacrificing the test 
quality. IJTAG allows for designing highly complex test networks compromising the latest test structure that 
enables to achieve the high test quality, as required by today's customers' applications. Besides the time overhead 
induced by the required network (re-)configuration, the access sequence of the instruments itself greatly affects 
the overall test time. Furthermore, the power characteristics of the complex test facilities have to be taken into 
account to avoid effects like IR drop during the later test application that is even more critical in highly multi- 
power domain networks. Consequently, the IJTAG methodology strictly requires an effective test scheduler that 
considers the individual instruments' constraints and is compatible with recent multi-power domain chip designs. 
This work proposes two novel power-aware test scheduling approaches based on pseudo-Boolean optimization 
and integer linear programming techniques that are both seamlessly integrated into a fully-automated test 
scheduling framework for IJTAG test networks even with multiple power domains. The first proposed optimi -
zation scheme allows for determining a local optimal test schedule, applicable to networks with more than one 
thousand instruments since the required run-time is manageable. Furthermore, the second optimization scheme 
determines a global optimal test scheduler, which is most suitable for mid-sized networks in which it is clearly 
outperforming any other existing technique.   
1.Introduction 
The state-of-the-art System-on-Chips (SoCs) embrace a large number 
of heterogeneous embedded components like digital-signal processors, 
sensors, communication and interface modules. The steadily increasing 
complexity of these chips has created a growing demand for effective 
infrastructures that enable efficient access methodologies for test, 
debug, and monitoring. Following this demand, IEEE 1687 Std. (IJTAG) 
has introduced new reconfigurable scan networks to tackle the challenge 
of long scan chains by adding programmable components such as 
Segment Insertion Bits (SIBs) and ScanMux Control Bits (SCBs) to the 
IJTAG test network (1). Despite the extensive test capabilities provided 
by this reconfigurable scan network, further technical challenges like IR 
drop (2,3) have to be addressed since, otherwise, the parallel test of all 
concurrently accessible instruments is impossible. The higher complexity of this problem in multi-power domain SOCs emphasizes the 
urgent need for more effective access management. 
In an IJTAG network, any on-chip module that holds only a client 
interface and is accessed by an external controller, is referred to as an 
instrument . The required configuration data for SIBs and SCBs is trans -
ferred serially along with the test data during every Capture-Shift-Update 
(CSU) cycle. In every CSU cycle, all instruments of the current active 
chain receive the new data from Scan-In (SI) port and at the same time 
shift their individual data to Scan-Out (SO) port of the chip. 
In order to eliminate the time penalty due to excessive configuration 
data, SIBs enable the creation of hierarchical scan chain designs. More 
precisely, SIBs can include or exclude a segment of the network that is 
connected to their host interface. This provides more efficient access to 
different parts of the SoC in comparison to the legacy daisy scan chains.

This paper heavily exploits formal techniques to develop a power- 
aware fully-automated test scheduling framework for IEEE 1687 
multi-power domain networks. At first, a Boolean Satisfiability (SAT)- 
based model is presented, which is applicable for arbitrary IJTAG test 
network architectures comprising SIBs, ScanMuxes, and SCBs. The 
optimization of test scheduling by considering multiple power domains 
is implemented by employing incremental Pseudo-Boolean Optimization 
(PBO) techniques. Various experiments clearly prove that this approach 
can be applied to networks containing over one thousand instruments 
without any concerns regarding the resulting CPU run-time. On top of 
this SAT-based approach, a novel Integer Linear Programming (ILP) model 
is proposed, which allows determining even global optimal test se-
quences for IJTAG test networks. The proposed models are processed by 
formal solving engines yielding power-safe tests and, hence, allowing for 
a significant test cost reduction.

the proposed framework as follows: Section 2 briefly reviews related 
works regarding the instrument access in IJTAG networks. Section 3 
provides a comprehensive description of the background that is needed

2.Related work 
Several techniques have been proposed to improve different aspects 
of the IJTAG networks (5–17). A hybrid scheduling scheme is presented 
in (6) to create a purely SIB-based network with minimized SIB pro-
gramming overhead. A new structure for SIB is proposed in (7) to in-
crease the bandwidth of the IJTAG network. This idea is further 
enhanced in (8) to create a scalable broadcast design that reduces the 
test time by sending identical data to the replicated cores. Other works 
like (11,18) have improved the access time without applying any 
structural modifications. In (9,19) , the authors have exploited the 
Boolean satisfiability problem to optimize the instrument access time for 
scan pattern retargeting in reconfigurable scan networks that do not 
include SIBs. This approach has been further improved by introducing 
an upper bound calculation method in (11). 
Due to the power, temperature, and other resource limitation, the 
simultaneous test of all concurrently accessible instruments is not 
feasible, and therefore the use of an instrument access scheduling scheme is inevitable. Optimization of the test time by using power 
constrained scheduling is investigated in (5,12,13) . A session-less test 
schedule about sorting the instrument based on the number of test 
patterns and resource conflicts is introduced in (12). The method pro-
posed in (13) uses a modified multi-bit SIB to follow a parallel sched -
uling approach based on an exhaustive search over all possible solutions. 
Typically, the embedded cores in the network require to be accessed a 
different number of times. The sequence and number of access in each 
test session considerably affect the overall test duration and, hence, an 
effective test scheduler has a major contribution to the reduction of final 
test cost. 
Generally, modern SoCs are divided into multiple power domains, 
which can be powered on or off selectively to manage the total power 
dissipation and avoid overheating (20). This important feature has not 
been considered in any test scheduler for IJTAG networks in the litera -
ture so far. However, for modern test schedulers, it is essential to 
consider the resulting power characteristics of the resulting test set, as 
being addressed by this work. Although IJTAG provides more flexible 
access to the instruments, the implementation of multiple power do-
mains along with the hierarchical structure of the network, non-linear 
constraints, and the existence of different concurrency groups as dis-
cussed in (14) poses new challenges against test scheduling. In work 
(14), a graph-based model of IJTAG networks has been proposed to 
detect the concurrently accessible instruments in multi-power domain 
networks. The access and power constraints are exploited to prune the 
search tree during the exhaustive search based on dynamic program -
ming. Despite the promising results, using this method in large networks 
requires intensive computations. 
In contrast to any other existing related work, this paper proposes a 
power-aware fully-automated test scheduler framework for IJTAG 
multi-power domain networks by heavily exploiting formal techniques 
while considering the instrument-specific power constraints to, in the 
end, determine highly effective but power-safe tests.

The IJTAG standard introduces two languages that are used to build 
up the underlying network structure and to reflect the required test data 
retargeting. First, the Instrument Connectivity Language (ICL) that de-
scribes the network structure on an abstract level without revealing the 
functional logic of the design. Secondly, the Procedural Description Lan-
guage (PDL) defining the sequence of operations, as required for 
accessing an instrument. In this paper, the ICL netlist is utilized to create 
a Boolean network description, which is subsequently used to analyze 
the network for obtaining an optimized access schedule. 
3.2. Boolean satisfiability 
The SAT problem attempts to determine whether a set of variable 
assignments of a given Boolean function exists such that the function is 
satisfied, i.e., is evaluated to TRUE . If such an assignment exists, the 
function is considered to be satisfiable (sat) and otherwise unsatisfiable 
(unsat) (21). A lot of research work has been spent on the development 
of powerful SAT solving algorithms yielding well-engineered SAT 
solvers like clasp (22), MiniSAT (23), SLIME (24), and ManySAT (25). 
This is since the SAT problem has recently been utilized to address 
various research questions effectively in domains like the structural test 
generation (26).

4.1. PBO-based scheduling model 
Both proposed test scheduling methods are applicable on arbitrary 
IJTAG networks comprising SIBs, ScanMuxes, SCBs, TDRs, and 
branching nodes. Every instrument requires a number of accesses that 
has to be fulfilled during the test process. The power consumption of 
each instrument, the number of power domains, and their power limits 
are considered as the input data to the scheduler. The inclusion or 
exclusion of instruments in/from a specific test session is well suitable 
for being mapped to the SAT problem, as done in this work. On top of 
this, the Boolean model is enriched by PB aspects to reflect the integer 
constraints regarding the power and number of accesses as well. Thus, 
the evaluation of this pseudo-Boolean function allows assessing these 
criteria of a determined test schedule. Finally, for not just assessing a test 
schedule but for determining a (local) optimal one, a PBO function ΨB
Gn→ℝ has been introduced, which allows optimizing a test schedule 
while considering the given power and access constraints. 
The proposed PBO-based scheduling approach can be divided into 
the following four main steps:  
1. Extracting the Boolean representation of the network from the ICL 
file. More precisely, three main IJTAG building blocks have been 
developed allowing for generating the overall SAT instance out of a 
given ICL description.  
2. Introducing the power constraints as PB components wit respect to 
the given instruments' power consumption and the predefined power 
domain limits. 
3.Adding the objective function to maximize the number of concur -
rently active instruments in every test session. 
4.The incremental invocation of the SAT solver to provide an opti-
mized access to the available instruments and subsequent masking of 
the fully covered instruments of currently optimized session to 
exclude them from the subsequent invocations. 
An overview of the proposed scheduling scheme is depicted in Fig. 3. 
It is assumed that seven instruments w1 to w7 which are shown as boxes, 
are required to be tested during the minimum possible time. This is 
achieved by organizing the boxes over a minimized number of strips 
which represent the CSUs. The instruments of Fig. 3 are divided into two 
power domains indicated with different colors with the power limits Pmax1 and Pmax2. The number of boxes equals the number of required 
access to the corresponding instrument. For example, three boxes of w4 
in Fig. 3 indicate three given accesses in the test pattern. The height of 
each box represents its power consumption. The horizontal axis is 
divided into CSU units. A set of successive CSUs, that contains the same 
selection of instruments, is called a (test) session, as indicated by S1 to S5 
in Fig. 3. For instance, the first three successive strips contain the same 
set of boxes forming the first session. The control signals are applied 
through the control circuitry during each CSU. These control signals are 
omitted from the Figures for a cleaner presentation. The corresponding 
control signals are created by the IJTAG-embedding EDA tool and driven 
by a controller. Thus, no dependencies between these signals exist over 
different sessions and CSUs. Every session in this scheme represents a 
scan chain that can be reordered due to their independence. The last CSU 
of every session contains the SIB configuration data required for creating 
the next scan chain. Over the scheduled strips, the sum of instruments' 
power consumption does not violate the corresponding power limit. This 
is shown with dashed and solid lines for domains j1 and j2, respectively. 
The power consumption of the scan chain elements are not considered 
since they are negligible compared with instruments' power consump -
tion. The proposed scheduler tries to allocate the boxes to the strips such 
that the number of introduced CSUs are minimized.

4.2. ILP-based optimization model 
In the previously proposed PBO-based method, the PBO solver de-
cides on the value of pure Boolean decision variables yielding a rather 
fast solving process. However, to create a global optimization scheme for 
the test scheduling, new integer decision variables are introduced to 
decide on the number of CSUs in each test session. This circumstance 
forms an integer linear programming problem that can be solved over a 
single run of an ILP solver, as described in Algorithm 2. The computa -
tional cost of this algorithm depends on three main factors. Assuming N 
as the number of instruments, the objective function will have N integer 
variables. Considering the Boolean variables for instruments, the solver 
determines the value of a total of N2 N decision variables. This search 
space is constrained by N ⋅ D equations defining the power constraint 
where D represents the number of power domains. The next factor that 
heavily constrains the search space concerns the inequalities that define 
the structural constraint of the given network. The number of in-
equalities grows linearly with the number of edges in the network's 
graph.
